How a Mobile Shop Website Works on the Internet

1. Working of the website

When a customer types www.durgmobileshop.com in the browser, the Domain Name System (DNS) works like an Internet phone book and converts the domain name into the IP address of the web server where the mobile shop website is stored.

After DNS finds the IP address, the browser sends an HTTP request to that server asking for the homepage of the shop. The server replies with HTML, CSS and JavaScript files. The browser reads these files and displays the page: a list of mobile phones, images, prices and “Add to Cart” buttons.

When the user clicks links or buttons, the browser may send more requests to the server, or JavaScript can update the page directly using the DOM. For example, clicking “View Details” shows extra information about one phone, and clicking “Add to Cart” updates the cart section with the selected mobile and total price.

2. Hosting the mobile shop website online

First the developer creates the website files (such as index.html, style.css and script.js) on the local computer and tests the site in a browser. To make it public, these files are uploaded to a web hosting service. For a simple static mobile shop, free hosting like GitHub Pages or Netlify can be used.

On GitHub Pages the steps are: create a repository, upload the mobile shop files, keep the main file name as index.html, and enable GitHub Pages in the repository settings. GitHub then gives a public URL such as https://username.github.io/mobile-shop/, and anyone with Internet access can open the shop on mobile phones or laptops using that link.

For a commercial shop with real payments, the site can be hosted on a professional web server. The shop owner registers a custom domain name, points the domain's DNS records to the hosting server, and enables HTTPS with an SSL certificate so that customer data is encrypted and secure while buying mobiles online.
